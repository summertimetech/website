The Web

Tooling vs Functies vs Architectuur

Zoals op 1e pagina van deze "beginners guide to development" aangegeven is tooling voor web presents onderverdeeld in vier categorien, Multimedia, CMS, Website Builders en Full Stack Website en App Development. Onderaan 1e pagina staat ook dat web presents mensenwerk is, en met die insteek is verdere structuur van deze guide opgebouwd. Dat wil zeggen wie doet wat en welke tooling wordt gebruikt, en op basis daarvan zijn functieprofielen opgemaakt

Functiesprofielen

1. Multimedia

Functie is Content Creator en bedient zich met tools die in staat stelt alle vormen van inhoud te vervaardigen. Dat kan zijn tekst, beeld, video etc.

2. Content management system (CMS)

Functie is Web Manager en bedient zich met "ready-for-use" CMS tools om content op het web te plaatsen

3. Website builder

Functie is Web Manager en bedient zich met "ready-for-use" Website Builder tools om content op het web te plaatsen

4. Full stack Website en App development

Functie is Full stack developer en bedient zich met tools om vanaf "scratch" Websites en Apps te maken op basis van aangeleverde, of zelfgemaakte, content. Bijvoorbeeld, deze technisch eenvoudige website van summertime.tech is vanaf "scratch" ontwikkeld obv HTML5 met CSS3 met teksteditor SublimeText. Scratch staat tussen haakjes want gebruik gemaakt van boilerplate Massively

Architectuur

De keuze van "welke tooling te gebruiken" om te komen tot “beste” web omgeving wordt ingegeven door hardware, software, tooling, beoogde content, functionaliteit, portabiliteit, performance, uiteindelijk geboden resultaat “het product”, mate van complexiteit, kosten en tijdsduur voor ontwikkelen, bouwen, beheren, life cycle management, mate van beschikbaarheid "tooling"-personeel want sommige web developers zijn sporadisch beschikbaar, kosten van dat personeel, opleidingen enzovoorts. Deze elementen vertalen zich naar een “architectuur keuze”. Oftewel een "blauwdruk" van het geheel van apparatuur, programmatuur en tooling voor web presents

Tooling in relatie tot architectuur

Deze guide heeft meerdere invalshoeken in zich. Want is te gebruiken als leidraad voor architecuur keuze, zoals welke apparatuur, welke programmatuur en welke tooling. Naast keuzegids is deze guide tevens te gebruiken als “handleiding” voor zelfstudie, met per tool verwijzing naar instructie video’s
